# OpenAI Long-Chat SmolLM2 1.7B Shared-Prefix Cache EOS Gate

## Scope

This run proves tokenizer-EOS `finish_reason=stop` behavior through Ferrite's
OpenAI-compatible streaming chat surface while the experimental shared-prefix
prompt cache is enabled. It uses the generated-context long-chat gate: turn 1
uses the seed assistant context, and turns 2-4 use assistant text generated by
prior completed streaming responses.

This is local evidence for one model and one EOS-sensitive prompt contract. It
does not prove EOS behavior across Qwen models, the 256/512/1024-token long-chat
matrix, x86_64, or longer steady-state serving.

## Server Command

```sh
target/release/ferrite-server \
  --model target/models/SmolLM2-1.7B-Instruct-Q4_K_M.gguf \
  --model-id SmolLM2-1.7B-Instruct-Q4_K_M \
  --bind 127.0.0.1:18180 \
  --api-key local-secret \
  --default-max-tokens 16 \
  --hard-max-tokens 64 \
  --inference-wait-ms 30000 \
  --experimental-prefix-cache
```

## Gate Command

```sh
target/release/ferrite-openai-long-chat-gate \
  --execute \
  --error-probe \
  --disconnect-probe \
  --addr 127.0.0.1:18180 \
  --api-key local-secret \
  --models SmolLM2-1.7B-Instruct-Q4_K_M \
  --token-lengths 16 \
  --turns 4 \
  --probe-max-tokens 16 \
  --rss-pid 63112 \
  --prompt 'Question: What is the capital of France? Answer only with the city name.' \
  --assistant-context 'Paris.' \
  --follow-up 'Question: What is the capital of France? Answer only with the city name.' \
  --expect-finish-reason stop \
  --prompt-cache-key long-chat:eos-answer-only-16 \
  --require-cached-follow-ups
```

The command exited successfully.

## Diagnostic Boundary

A naive repeated completion prompt did not produce EOS for generated follow-up
turns, even though cache reuse was present. The diagnostic run used:

```text
--prompt 'The capital of France is'
--assistant-context ' Paris.'
--follow-up 'The capital of France is'
--token-lengths 64
--prompt-cache-key long-chat:eos-diagnose-64
```

It produced:

```text
turn 1: finish_reason=stop, cached_prompt_tokens=0, completion_tokens=4
turn 2: finish_reason=length, cached_prompt_tokens=14, completion_tokens=64
turn 3: finish_reason=length, cached_prompt_tokens=14, completion_tokens=64
turn 4: finish_reason=length, cached_prompt_tokens=14, completion_tokens=64
long_chat_summary_cached_generated_follow_up_turns=3
long_chat_summary_run_complete=true
```

That boundary matters: cache reuse does not imply EOS behavior. The prompt
contract must be EOS-sensitive in the multi-turn chat template.

## Interpretation

Ferrite's shared-prefix cache can coexist with natural tokenizer EOS
termination on the OpenAI-compatible streaming chat path for
`SmolLM2-1.7B-Instruct-Q4_K_M`. The run proves that generated-context follow-up
turns were cached, terminated via `finish_reason=stop` without an explicit
OpenAI `stop` parameter, preserved usage accounting, emitted token-id summaries,
sampled RSS, and recovered after unauthorized and disconnect/reconnect probes.

This closes a small cached-EOS proof slice only. Broader EOS coverage remains
open for Qwen models, x86_64, larger prompt contracts, longer steady-state
serving, and the full long-chat milestone.
