# 2026-06-27 Tier 1 x86_64 AVX2 Q6_K Matvec

## Slice

After F32, Q8_0, and Q5_0 AVX2 compile bring-up, this slice adds x86_64 AVX2
compile coverage for Q6_K matrix-vector multiplication while preserving scalar
fallback and the existing aarch64 NEON path.

The implementation:

- adds an `X86_64Avx2` Q6_K matvec backend behind `target_arch = "x86_64"`;
- dispatches to AVX2 only after `is_x86_feature_detected!("avx2")`;
- keeps Q6_K AVX2 dispatch scoped to non-zero column counts that are whole
  256-value K-blocks per row;
- reuses the existing Q6_K block decoder; and
- keeps the unsafe intrinsic use inside the `scalar/q6_k.rs` x86_64 module.

## Validation

Test-first failure before implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
```

The check failed because the x86_64-only test referenced
`Q6KMatVecBackend::X86_64Avx2`, which did not exist.

Passing checks after implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
cargo test -p ferrite-inference q6_k_matvec_uses_neon_backend_on_aarch64 -- --nocapture
cargo test -p ferrite-inference --test matvec_kernel_check -- --nocapture
cargo fmt --all -- --check
git diff --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

The x86_64 target check compiled the AVX2 Q6_K backend and x86_64-only backend
test without warnings. The local aarch64 NEON Q6_K backend test passed, the
matvec reference-check target passed all 6 tests, and the full workspace test
suite passed.

## Remaining Work

This is compile-time AVX2 bring-up only. It does not prove AVX2 runtime
correctness on an x86_64 host, does not add AVX2 Q4_K, and does not provide
Tier 1 throughput evidence.
