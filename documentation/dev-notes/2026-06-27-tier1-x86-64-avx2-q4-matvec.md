# 2026-06-27 Tier 1 x86_64 AVX2 Q4_K Matvec

## Slice

This slice completes compile-time x86_64 AVX2 matvec bring-up for the currently
supported F32 and quantized matrix formats by adding Q4_K coverage. Runtime
correctness on x86_64 hardware remains unproven.

The implementation:

- adds an `X86_64Avx2` Q4_K matvec backend behind `target_arch = "x86_64"`;
- dispatches to AVX2 only after `is_x86_feature_detected!("avx2")`;
- keeps Q4_K AVX2 dispatch scoped to non-zero column counts that are whole
  256-value K-blocks per row;
- reuses the existing Q4_K block decoder; and
- keeps the unsafe intrinsic use inside the `scalar/q4_k.rs` x86_64 module.

## Validation

Test-first failure before implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
```

The check failed because the x86_64-only test referenced
`Q4KMatVecBackend::X86_64Avx2`, which did not exist.

Passing checks after implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
cargo test -p ferrite-inference q4_k_matvec_uses_neon_backend_on_aarch64 -- --nocapture
cargo test -p ferrite-inference --test matvec_kernel_check -- --nocapture
cargo fmt --all -- --check
git diff --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

The x86_64 target check compiled the AVX2 Q4_K backend and x86_64-only backend
test without warnings. The local aarch64 NEON Q4_K backend test passed, the
matvec reference-check target passed all 6 tests, and the full workspace test
suite passed.

## Remaining Work

This is compile-time AVX2 bring-up only. It does not prove AVX2 runtime
correctness on an x86_64 host and does not provide Tier 1 throughput evidence.
