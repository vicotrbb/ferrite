# 2026-06-27 Tier 1 x86_64 AVX2 Q8_0 Matvec

## Slice

After the F32 AVX2 compile bring-up, this slice adds the first x86_64 AVX2
quantized matvec boundary for Q8_0 matrices while preserving scalar fallback and
the existing aarch64 NEON path.

The implementation:

- adds an `X86_64Avx2` Q8_0 matvec backend behind `target_arch = "x86_64"`;
- dispatches to AVX2 only after `is_x86_feature_detected!("avx2")`;
- widens eight signed Q8_0 values at a time to F32 lanes with AVX2 intrinsics;
- applies the existing per-block F16 scale after each block dot; and
- keeps the unsafe intrinsic use inside the `scalar/q8_0.rs` x86_64 module.

## Validation

Test-first failure before implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
```

After fixing the x86_64 test import setup, the check failed because the
x86_64-only test referenced `Q8_0MatVecBackend::X86_64Avx2`, which did not
exist.

Passing checks after implementation:

```text
cargo check -p ferrite-inference --target x86_64-unknown-linux-gnu --tests
cargo test -p ferrite-inference q8_0_matvec_uses_neon_backend_on_aarch64 -- --nocapture
cargo test -p ferrite-inference --test matvec_kernel_check -- --nocapture
cargo fmt --all -- --check
git diff --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

The x86_64 target check compiled the AVX2 Q8_0 backend and x86_64-only backend
test without warnings. The local aarch64 NEON Q8_0 backend test passed, the
matvec reference-check target passed all 6 tests, and the full workspace test
suite passed.

## Remaining Work

This is compile-time AVX2 bring-up only. It does not prove AVX2 runtime
correctness on an x86_64 host, does not add AVX2 Q5_0, Q4_K, or Q6_K kernels,
and does not provide Tier 1 throughput evidence.
